## 0. 설계 원칙 (재확인)

1. **복잡함을 덜어내고 본질에 집중한다** — 한 화면의 유료화 메시지는 1순위 1개만 강조한다.
2. **망설일 순간을 주지 않는다** — 결정 지점에서 다음 행동을 기본값으로 제시한다.
3. **헷갈리게 하지 않는다** — 유료 전환 유도 패턴은 **3가지 이하**로 통일한다.
   - **A. 진입 불가 (잠금)** — 가치 상상이 어렵고 복잡도가 높은 기능
   - **B. 진입 가능 + 상단 띠배너** — 기능을 직접 둘러볼 때 가치 체감 효과가 큰 기능
   - **C. 사용 중 제한 (넛지 모달)** — 무료 할당량을 소진한 경우

> ⚠️ 기존 문서의 "설명 화면A(AI) / 설명 모달B(통계) / 진입 띠배너 / 넛지 모달 / 클릭 시 체험 제안 모달" 5종은 위 A/B/C로 수렴합니다.

## 1. 공통 컴포넌트

### 1.1 할당량 인디케이터 (Quota Indicator)

**역할** 남은 무료 할당량을 상시 표시하여 유료 전환 타이밍을 인지시킨다.

**위치** 리뷰 목록 상단, LNB 하단(축약형)

**구성**
- 라벨: "이번 달 무료 리뷰 요청"
- 숫자: `사용량 / 총량` (예: 18 / 30)
- 프로그레스 바 (4단계 상태 색 적용)
- 보조 텍스트 (상태별로 변경, 아래 상태 정의 참조)

**상태**

| 상태 | 조건 | 바 색상 | 보조 텍스트 예시 |
|---|---|---|---|
| Safe | 0–59% 사용 | Neutral | `잔여 12건` |
| Warn | 60–89% 사용 | Warning | `잔여 5건 · 곧 소진돼요` |
| Critical | 90–99% 사용 | Accent | `잔여 1건 · 지금 체험 시작하기` |
| Exhausted | 100% 사용 | Disabled 배경 + Accent 아이콘 | `이번 달 할당량 소진 · 체험 시작하기` |

**인터랙션**
- Warn 이상 상태에서 인디케이터 전체 영역 클릭 가능
- 클릭 시 → `넛지 모달 C (무료 체험 제안)` 오픈
- Safe 상태에서는 클릭 비활성 (단순 정보 표시)

---

### 1.2 상단 띠배너 (Top Banner)

**역할** "지금 이 화면은 유료 기능 미리보기"임을 인지시킨다.

**위치** 페이지 콘텐츠 최상단, LNB 우측, 스크롤 시 고정(Sticky)

**구성**
- 아이콘 (자물쇠, 16px)
- 메시지 (좌측, 예: `유료 기능을 미리 보고 있어요. 14일 무료로 실제 사용해보세요.`)
- CTA 버튼 (우측, Primary, `14일 무료 체험`)
- 닫기 버튼 (X, 세션 기준 숨김)

**상태**
- Default: 배경 Accent Soft
- Hover on CTA: Primary Dark
- Dismissed: 세션 내 숨김, 다음 로그인 시 재노출

**원칙**
- 띠배너가 있는 화면에서는 기능 내부에 **추가 유료 안내를 중복 배치하지 않는다**
- 높이는 **40px 고정**, 세로 공간 점유 최소화

---

### 1.3 프리미엄 잠금 표시 (Lock Indicator)

**역할** LNB·메뉴·기능 카드에서 유료 기능임을 시각적으로 구분한다.

**구성**
- 자물쇠 아이콘 (12px, Neutral 60)
- 텍스트 우측에 위치, 간격 4px

**인터랙션**
- 클릭 시 → `기능 설명 모달 (B)` 오픈
- Hover 시 Tooltip 노출: `유료 기능 — 클릭해서 알아보기`

## 2.1 온보딩 (무료 버전 전용)

### 목적
카드 미등록 상태에서도 설치를 완료시켜, 유저가 **"둘러보기 → 가치 체감 → 체험 시작"** 순서로 진입하도록 한다.

### 전체 플로우
```
[스텝 1] 샵 연동 → [스텝 2] 기본 정보 입력 → [스텝 3] 위젯 설치 안내
→ [스텝 4] 무료 버전 시작 (=설치 완료) → [리뷰 목록 랜딩]
   └ 카드/알림톡 등록은 이후 "리뷰 요청" 시점에 인라인으로 유도
```

> **변경점** As-Is: 카드 등록 전엔 설치 불가 → To-Be: 온보딩 종료 시 즉시 설치 완료

### 스텝별 레이아웃

#### Step 1. 샵 연동
- 중앙 카드(Width 480px)
- 샵 플랫폼 선택 (카페24, 쇼피파이, 스마트스토어 등)
- 하단 진행률 바 (1/4)

#### Step 2. 브랜드 기본 정보
- 샵 이름, 대표 색상 1종, 로고 업로드 (선택)
- Skip 허용 (`나중에 설정할게요` — Tertiary Button)

#### Step 3. 위젯 설치 안내
- 좌측: 설치 가이드 (3스텝 체크리스트)
- 우측: 실시간 설치 상태 확인 (샵에서 감지되면 자동 체크)
- CTA: `설치 건너뛰고 먼저 둘러보기` (Secondary) + `설치 확인` (Primary)

#### Step 4. 무료 버전 시작 (완료 화면)
- 타이틀: `준비됐어요. 지금부터 알파리뷰를 둘러보세요.`
- 서브: `무료 버전에서는 30건의 리뷰 요청을 체험할 수 있어요.`
- Primary CTA: `리뷰 목록으로 이동`
- Secondary CTA: `카드 등록하고 알림톡 바로 받기` (선택 유도, 강요 X)

### 상태
- 각 스텝 이탈 시 진행 상황 자동 저장 (재진입 시 해당 스텝 복원)
- 스텝 3에서 5분간 위젯 감지 실패 → 건너뛰기 버튼을 Primary로 승격

### 엣지 케이스
- 샵 연동 실패 → 인라인 에러 + 지원팀 연결 링크
- 이미 유료 유저가 다른 계정으로 온보딩 진입 → 경고 모달 + 기존 계정 복귀 유도

### 오픈 질문
- ☐ 카드 등록을 온보딩에 선택지로 남길지, 아예 빼고 "리뷰 요청" 시점에만 노출할지? (단순성 관점에서는 후자 권장)

## 2.2 리뷰 목록 (할당량 · 현황)

### 목적
무료 유저의 **메인 화면**. 유료 전환 동기는 이 화면에서 자연스럽게 생긴다.

### 레이아웃 (상단 → 하단)

```
┌─────────────────────────────────────────────┐
│ (A) 할당량 인디케이터 + 리뷰 현황 요약 카드   │
├─────────────────────────────────────────────┤
│ (B) 필터/검색 툴바                           │
├─────────────────────────────────────────────┤
│ (C) 리뷰 테이블                              │
├─────────────────────────────────────────────┤
│ (D) 페이지네이션                             │
└─────────────────────────────────────────────┘
```

### (A) 상단 요약 영역 — **핵심**

2개 컬럼(Width 각 50%)으로 구성. **한 화면에 넛지 메시지가 중복되지 않도록** 요약 영역에만 유료 관련 정보를 집중한다.

#### 좌측: 할당량 인디케이터
- 공통 컴포넌트 1.1 사용
- 상태에 따라 CTA 톤 조정

#### 우측: 리뷰 현황 카드
- 지표 2개 (핵심만)
  1. **이번 달 리뷰 작성률** (퍼센트 + 지난달 대비 증감)
  2. **리뷰 작성 후 구매 전환율** (퍼센트 + 증감)
- 카드 하단에 작은 링크: `알파리뷰 고객 평균과 비교하기 →` (클릭 시 상세 뷰)

> **원래 문서와의 차이**: 놓친 개수, 예상 효과 %, A/B 변화율을 모두 동시에 보여주려 했으나 → **핵심 2지표만 남기고 나머지는 "상세 보기" 뷰로 분리**. 원칙 1(본질 집중)에 맞추기 위함.

### (B) 필터/검색 툴바
- 검색창, 기간 필터, 별점 필터, 정렬
- 무료 버전에서 비활성 항목 없음 (필터는 전부 허용)

### (C) 리뷰 테이블
- 컬럼: 작성일 / 상품 / 별점 / 리뷰 내용 / 작성자 / 채널 / 액션
- 포인트 지급 컬럼은 무료 버전에서 **숨김** (유료 전환 시 노출)
- 액션 컬럼: 댓글 달기(자물쇠), 뱃지 부여(자물쇠) → 클릭 시 `기능 설명 모달 B`

### (D) 페이지네이션
- 기본 25건/페이지, 최대 100건/페이지

### 상태별 변화

| 할당량 상태 | (A) 좌측 | (A) 우측 | 리뷰 요청 버튼 |
|---|---|---|---|
| Safe | 기본 인디케이터 | 현황 카드 | `리뷰 요청 보내기` 활성 |
| Warn | Warning 컬러 | 현황 카드 | 버튼 옆에 `남은 5건` 캡션 |
| Critical | Accent + 맥동 애니메이션 | 현황 카드 | Caption → Accent 색상 |
| Exhausted | CTA 형태로 변환 (`14일 체험 시작`) | 현황 카드 (변화 없음) | 버튼 비활성 + 클릭 시 넛지 모달 |

### 엣지 케이스
- 리뷰 0건: 테이블 영역이 Empty State로 대체 (`아직 리뷰가 없어요. 첫 리뷰 요청을 보내볼까요?`)
- 할당량 초과 시 추가 리뷰 요청 시도 → `넛지 모달 C` 오픈

### 오픈 질문
- ☐ "고객 평균과 비교" 지표는 유료 유저에게도 동일하게 노출할지, 무료 넛지 전용일지?
- ☐ 리뷰 작성 후 구매 전환율 — 데이터 충분성(N≥30) 미달 시 숨김 규칙 필요

## 2.3 LNB + 띠배너

### 목적
메뉴 계층에서 "무엇이 유료인지"를 한눈에 보여주되, **자물쇠를 너무 많이 노출하지 않는다** (자물쇠 오염 방지).

### LNB 구조 (무료 버전)

```
─── 대시보드 🏠
─── 리뷰 목록
─── 리뷰 수집
      ├─ 스태프 리뷰
      ├─ 인스타 & 블로그
      ├─ 스마트 스토어
      ├─ 리뷰 요청         🔒  ← Lock
      ├─ SNS 리뷰 챌린지   🔒  ← Lock
      └─ 체험단 리뷰       🔒  ← Lock
─── 디자인 설정
      ├─ 공통 디자인
      ├─ 리뷰 상세 화면
      ├─ 리뷰 작성 유도
      └─ 위젯 설정         (일부 자물쇠 — 세일즈티커, 키워드 위젯)
─── 기본 설정
─── ─────────────
─── AI 리뷰 분석         🔒  ← 대분류 통째 잠금
─── 통계                  🔒  ← 대분류 통째 잠금
─── ─────────────
─── [🎁 14일 무료 체험 시작하기]  ← LNB 하단 고정 CTA
```

### 잠금 규칙
- **대분류 통째로 잠근 경우** (AI 리뷰 분석, 통계): 자물쇠를 대분류에만 표시, 하위 메뉴는 확장하지 않음 → **자물쇠 노출 최소화**
- **혼합 영역** (리뷰 수집, 디자인 설정): 중분류 확장 후 소메뉴별 자물쇠 표시
- 자물쇠 메뉴 클릭 → **진입하지 않고** `기능 설명 모달 B` 오픈 (유저가 어디 있는지 혼란 방지)

### 상단 띠배너 노출 규칙

| 진입 화면 | 띠배너 표시 | CTA |
|---|---|---|
| 무료 버전 기본 제공 화면 (리뷰 목록, 온보딩 등) | 표시 안 함 | — |
| 유료 기능 둘러보기 화면 (리뷰 뱃지, 세일즈티커 설정 등) | 표시 | `14일 무료 체험` |
| 사용량 제한된 무료 기능 (할당량 초과 상태) | 표시 | `14일 무료 체험` |

### 상태
- LNB 접힘/펼침 상태는 로컬 스토리지 기억
- 자물쇠 메뉴에 Hover 시 Tooltip (`유료 기능 — 클릭해서 알아보기`)

### 엣지 케이스
- 유료 → 무료 버전 다운그레이드 유저가 기존에 사용하던 유료 메뉴에 진입 시도 → 진입 허용하되 **상단 띠배너 + 데이터 보존 안내**:
  `설정은 유지되지만 무료 버전에서는 편집이 제한돼요. 재구독 시 그대로 복원됩니다.`

## 2.4 넛지 모달 3종

### 공통 사양
- 너비 480px, 중앙 정렬
- 닫기 X (우상단), 바깥 영역 클릭 시 닫힘
- **모달 제목 → 시각(프리뷰 or 지표) → 본문 → CTA 2개** 순서 고정
- 톤은 **"제안"** 이지 **"차단 경고"** 가 아니다 (느낌표, 빨간 경고 배경 지양)

---

### 모달 A — 기능 설명 (AI·통계 통합)

**트리거** LNB 자물쇠 메뉴 클릭, 기능 카드 내부 잠금 버튼 클릭

**구성**
- 상단: 기능 썸네일 (16:9, 실제 기능 스크린샷 or 애니메이션 GIF)
- 제목: `{기능명} — {한 줄 가치 제안}`
  - 예: `월간 리뷰 브리핑 — 이번 달 고객이 뭘 좋아했는지 AI가 요약해드려요`
- 본문: 3개 이내의 Bullet (핵심 가치만)
- CTA
  - Primary: `14일 무료로 사용하기`
  - Tertiary: `둘러보기만 할게요` → 실제 기능 화면으로 진입 (유료 기능 중 일부만 허용, 표 참조)

**둘러보기 허용 여부**

| 기능 분류 | 둘러보기 | 근거 |
|---|---|---|
| AI 리뷰 분석 전체 | ❌ 불가 | 복잡도 높음, 빈 데이터 화면이 오히려 가치 저해 |
| 통계 전체 | ❌ 불가 | 실데이터 없이는 가치 체감 불가 |
| 리뷰 뱃지 / 댓글 | ✅ 가능 | 설정 화면만 봐도 기능 이해 가능 |
| 세일즈 티커 / 키워드 위젯 | ✅ 가능 | 위젯 프리뷰로 가치 체감 |

> **단순화 포인트**: 원래 "설명 화면A"와 "설명 모달B"로 나뉘어 있던 것을 **모달 1종 + 둘러보기 가능 여부 플래그**로 통일.

---

### 모달 B — 사용량 제한 도달

**트리거** 무료 할당량 100% 소진 후 해당 기능을 다시 시도

**구성**
- 상단: 심플한 일러스트 (게이지 가득 찬 모습)
- 제목: `이번 달 무료 할당량을 모두 사용했어요`
- 본문
  - `14일 무료 체험으로 {기능명}을 제한 없이 사용해보세요.`
  - 캡션: `체험 종료 후 결제 전 이메일로 알려드려요` (불안 해소)
- CTA
  - Primary: `14일 무료 체험 시작하기`
  - Tertiary: `다음 달까지 기다릴게요`

**주의**
- 모달 내부에 **놓친 개수, 예상 %, A/B 비교** 정보를 중복 배치하지 않는다 (상단 인디케이터가 이미 전달)
- 체험 시작 플로우는 **카드 등록 인라인 화면**으로 연결

---

### 모달 C — 무료 체험 제안 (맥락 진입용)

**트리거**
- SNS 챌린지, 체험단 등 "진입 불가" 기능 클릭
- 할당량 인디케이터의 CTA 버튼 클릭

**구성**
- 상단: 기능 프리뷰 (모달 A보다 더 크고 감각적인 비주얼)
- 제목: `{기능 카테고리}로 리뷰 수집을 확장해보세요`
- 본문
  - 핵심 가치 2줄
  - **예상 효과 1지표만** (예: `평균 리뷰 작성률 +37% ↑`) — 원칙 1 적용, 여러 수치 나열 금지
- 요금제 요약 (1줄, 예: `월 {금액}원부터 · 언제든 해지 가능`)
- CTA
  - Primary: `14일 무료 체험 시작하기`
  - Secondary: `요금제 자세히 보기`
  - Tertiary: `나중에 할게요`

## 3. 전환 트리거 매트릭스

유저가 마주치는 각 상황에서 **어떤 패턴**이 발동하는지 한 테이블로 통일한다.

| 상황 | 패턴 | 컴포넌트 |
|---|---|---|
| 유료 대분류 클릭 (AI, 통계) | A. 진입 불가 | 모달 A (둘러보기 불가) |
| 유료 소분류 클릭 (리뷰 뱃지 등) | B. 진입 + 띠배너 | 띠배너 + 실기능 UI |
| 특수 수집 기능 클릭 (SNS, 체험단) | A. 진입 불가 | 모달 C |
| 무료 기능 할당량 소진 후 재시도 | C. 사용 중 제한 | 모달 B |
| 할당량 인디케이터 CTA 클릭 | C. 직접 유도 | 모달 C |

## 4. 공통 카피 톤 가이드

- **"유료"** 라는 단어 대신 `14일 무료 체험`을 기본으로 사용 (체험 가능성을 먼저 인지)
- 느낌표 금지, 명령형 금지 (예: ❌ `지금 바로 시작하세요!` → ✅ `14일 무료 체험 시작하기`)
- 불안 유발 금지 (예: ❌ `놓친 리뷰 N개` → ✅ `이번 달 리뷰 요청 N건 남음`)
- 체험 종료 알림 약속을 항상 병기 (결제 불안 해소)

## 5. 오픈 이슈 (의사결정 필요)

1. ☐ 무료 → 유료 다이렉트 전환 경로 허용 여부 (현재 문서는 무료체험 종료 후에만 가능)
2. ☐ 유료 → 무료 다운그레이드 시 데이터 보존 기간 및 정책 공지 UI 위치
3. ☐ "리뷰 작성 후 구매 전환율" 지표의 최소 데이터량 기준
4. ☐ AI/통계에서 "맛보기" 한 조각 제공 가능성 (월간 브리핑 샘플 1건 등) — 가치 체감 강화용
5. ☐ 알림톡 카드 등록 유도 시점 — 온보딩 Step 4 vs 최초 리뷰 요청 시점
6. ☐ LNB 하단 고정 CTA의 A/B 테스트 (상시 노출 vs 스마트 노출)
